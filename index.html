What this tutorial is

If you have run a distributed system, you know the failure mode: a request crosses half a dozen services, and when it is slow or quietly wrong, the logs you have are too sparse to explain it, too noisy to search, or costly enough that you hesitate to add more. The thing that would actually help — the request's full path, in order, across every service, with the time spent in each hop and the cost of serializing data between them — is the thing you almost never have.

This tutorial builds it. A single order request fans out across REST, gRPC, GraphQL, Kafka, and Postgres, and chapter by chapter you instrument it with OpenTelemetry until that one request is a single trace you can read end to end — with the metrics that show it in aggregate and the logs that explain it correlated to the same trace, all in a Grafana stack you run yourself. It stays honest about the trade-offs throughout: what auto-instrumentation gives you for free, where you pay for it in overhead or noise, and when one hand-placed span is worth more than a dozen automatic ones. The six services (order, inventory, payment, shipping, notification, review) are a realistic stand-in for systems you already operate; the subject is the observability, and what it costs to get it.
