What this tutorial is

A chapter-based, hands-on path to making a realistic Python service mesh observable with OpenTelemetry. A single order request fans out across an external REST API, service-to-service gRPC and GraphQL, asynchronous Kafka events, and Postgres — and you make the whole journey show up as one correlated trace, with metrics and logs that link back to it, in a self-hosted Grafana stack. The domain objects mirror the data-mesh reference architecture (order, inventory, payment, shipping, notification, review). It assumes working knowledge of Python and these technologies — it is not a tutorial on any of them. Everything runs locally in Podman; all images are UBI-based. This is an OpenTelemetry talk, not a Kubernetes one.
